Step 3: Sentiment Analysis

We'll perform sentiment analysis on the review texts using a pre-trained DistilBERT model fine-tuned for sentiment classification. The sentiment will be categorized as positive or negative.

In[]:
# Load pre-trained DistilBERT model and tokenizer for sentiment analysis
model_name = "distilbert-base-uncased-finetuned-sst-2-english"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = TFAutoModelForSequenceClassification.from_pretrained(model_name)

# Create a sentiment analysis pipeline
sentiment_pipeline = pipeline("sentiment-analysis", model=model, tokenizer=tokenizer, framework="tf", device=0 if device_name else -1)

# Apply the sentiment analysis pipeline to the review texts
df['sentiment'] = df['review_text'].apply(lambda x: sentiment_pipeline(x)[0]['label'])
df['sentiment_score'] = df['review_text'].apply(lambda x: sentiment_pipeline(x)[0]['score'])

# Display the sentiment analysis results

Step 7: Topic Modeling

In this step, we'll perform topic modeling on the review texts using Latent Dirichlet Allocation (LDA). This will help us identify common themes or topics discussed in the reviews.

In[]:
from sklearn.feature_extraction.text import CountVectorizer
from sklearn.decomposition import LatentDirichletAllocation

# Vectorize the text data
vectorizer = CountVectorizer(stop_words='english', max_features=1000)
X = vectorizer.fit_transform(df['review_text'])

# Fit the LDA model
lda = LatentDirichletAllocation(n_components=5, random_state=42)
lda.fit(X)

# Display the top words in each topic
def display_topics(model, feature_names, no_top_words):
    topics = {}
    for topic_idx, topic in enumerate(model.components_):
        topics[f'TOPIC {topic_idx + 1}'] = [feature_names[i] for i in topic.argsort()[-no_top_words:]]
    return topics

# Number of top words to display
no_top_words = 10

# Display the topics
topics = display_topics(lda, vectorizer.get_feature_names_out(), no_top_words)
for topic, words in topics.items():
    print(f"{topic}: {', '.join(words)}")

Conclusion and Insights

After analyzing the TikTok app review data, several key insights emerged that can inform product development, user engagement strategies, and overall app performance:

1. Sentiment Analysis

  • Overall Sentiment: The sentiment analysis revealed that a significant portion of the reviews are positive. This indicates that users generally have a favorable opinion of the app.
  • Negative Sentiment: While the majority are positive, there is a notable number of negative reviews, which often highlight issues such as bugs, user interface frustrations, and content moderation concerns. Addressing these could improve user satisfaction.

2. App Version Analysis

  • Review Distribution: The distribution of reviews across different app versions shows that newer versions generally receive more reviews. This could be due to increased user engagement with updated features or a growing user base.
  • Sentiment by Version: By analyzing the sentiment across app versions, it was observed that some versions have higher proportions of negative reviews, potentially indicating issues introduced in those updates. Monitoring sentiment after each app release can help in identifying and quickly addressing problems.

3. Upvote Distribution

  • User Engagement: The distribution of upvotes, which represents user engagement, shows that while most reviews receive a moderate number of upvotes, a few reviews stand out with significantly higher engagement. This might be due to the relevance or relatability of these reviews to other users.
  • Sentiment Correlation: The analysis shows a correlation between sentiment and the number of upvotes, with positive reviews generally receiving more upvotes. This suggests that satisfied users are more likely to engage with the community by endorsing content they agree with.

4. Trend Analysis

  • Review Trends Over Time: The trend analysis indicates how the volume of reviews and the average sentiment score changes over time. Spikes in review activity often coincide with new app releases or significant updates.
  • Sentiment Fluctuations: Changes in the average sentiment score over time may reflect user response to specific updates or changes in app policy. Understanding these trends can help in predicting and preparing for user feedback following major updates.

5. Topic Modeling

  • Common Themes: Topic modeling uncovered several recurring themes in the reviews. Common topics include user interface experience, content creation features, and issues with app performance. Understanding these themes can help prioritize improvements in areas most frequently discussed by users.
