Customer Churn Prediction for Loblaws Digital

Objective

The primary objective of this project is to develop a predictive model that accurately identifies customers at risk of leaving the company (churning) based on various customer-related features. By predicting churn, Loblaws Digital can take proactive measures to retain these customers, thereby reducing the overall churn rate and improving customer loyalty.

    • Import the dataset and inspect its structure and contents.

  2. Data Preprocessing:

  3. Exploratory Data Analysis (EDA):

  4. Feature Selection:

  5. Model Building:

    • Develop and evaluate different machine learning models, starting with a baseline logistic regression model and progressing to more complex models like Random Forest and XGBoost.

  6. Model Evaluation:

    • Compare the performance of different models using metrics such as accuracy, precision, recall, and F1-score. Identify the best-performing model for predicting customer churn.

  7. Model Tuning and Optimization:

    • Fine-tune the best-performing model to further enhance its predictive capabilities.

  8. Conclusions

    • Summarize findings, provide insights into the most influential features.

Dataset Overview

The dataset includes various features related to customer behavior, demographics, and interaction with Loblaws Digital. The key features include:

  • CustomerID: Unique identifier for each customer.
  • Churn: Target variable indicating whether a customer has churned (1) or not (0).
  • Tenure: Length of time the customer has been with the company.
  • PreferredLoginDevice, PreferredPaymentMode, Gender, etc.: Categorical features describing customer preferences and demographics.
  • Numerical Features: Features such as WarehouseToHome, OrderAmountHikeFromlastYear, and CashbackAmount, which provide insights into customer activity.

This notebook will guide you through the entire process, from data loading to building a robust predictive model for customer churn, ultimately empowering Loblaws Digital to make data-driven decisions to enhance customer retention.

Step 2: Data Preprocessing - Handling Missing Values

In this step, we will handle missing values in the dataset. We observed earlier that some columns like Tenure and HourSpendOnApp have missing values. We'll impute these missing values using appropriate strategies:

  1. Numerical Columns: We'll use the median of the respective column to fill in missing values.
  2. Categorical Columns: We'll use the most frequent category (mode) to fill in missing values.
In[]:
# Handling missing values

# For 'Tenure' and 'HourSpendOnApp', we will fill missing values with the median
df['Tenure'] = df['Tenure'].fillna(df['Tenure'].median())

Step 6: Feature Selection and Model Building

In this step, we will:

  1. Select Features: Use all available features for the initial model.
  2. Split the Data: Divide the dataset into training and testing sets to evaluate the model's performance.
  3. Build a Baseline Model: Use logistic regression as a baseline model to predict customer churn.

The following code block performs these tasks:

In[]:
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score, classification_report

# Features and target variable
X = df_encoded.drop(columns=['CustomerID', 'Churn'])
y = df_encoded['Churn']

# Split the data into training and testing sets
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# Initialize and train the logistic regression model
model = LogisticRegression(max_iter=1000)
model.fit(X_train, y_train)

# Predict on the test set
y_pred = model.predict(X_test)

# Evaluate the model
accuracy = accuracy_score(y_test, y_pred)
classification_report_output = classification_report(y_test, y_pred)

# Output the accuracy and classification report
print("Accuracy: ", accuracy)
print('Classification report:\n',classification_report_output)

Analysis of Model Performance

1. Accuracy:

  • The logistic regression model achieved an accuracy of 89.99%, which is quite high. However, accuracy alone does not provide a complete picture, especially in cases with class imbalance (as we have with churn prediction).

2. Classification Report:

Step 7: Exploring Alternative Models to Improve Prediction

In this step, we will explore more sophisticated models to improve the prediction of customer churn, especially for the minority class (churners). We will use the following models:

  1. Random Forest Classifier
  2. Gradient Boosting Classifier (XGBoost)

We will train both models and compare their performance to determine which one better predicts customer churn.

The following code block implements these models:

In[]:
from sklearn.ensemble import RandomForestClassifier
from xgboost import XGBClassifier
from sklearn.metrics import accuracy_score, classification_report

# Initialize the models
random_forest = RandomForestClassifier(random_state=42, class_weight='balanced')
xgb = XGBClassifier(random_state=42, scale_pos_weight=len(y_train)/sum(y_train))  # Handles imbalance

# Train the Random Forest model
random_forest.fit(X_train, y_train)
y_pred_rf = random_forest.predict(X_test)

# Train the XGBoost model
xgb.fit(X_train, y_train)
y_pred_xgb = xgb.predict(X_test)

# Evaluate the Random Forest model
accuracy_rf = accuracy_score(y_test, y_pred_rf)
classification_report_rf = classification_report(y_test, y_pred_rf)

# Evaluate the XGBoost model
accuracy_xgb = accuracy_score(y_test, y_pred_xgb)
classification_report_xgb = classification_report(y_test, y_pred_xgb)

# Output the results\
print("Random forest")
print("Accuracy: ", accuracy_rf)
print('Classification report:\n',classification_report_rf)

print("\n\nXGBoost")
print("Accuracy: ", accuracy_xgb)
print('Classification report:\n',classification_report_xgb)
